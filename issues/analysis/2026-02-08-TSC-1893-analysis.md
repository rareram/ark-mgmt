# 1. [개발팀 보고] 기술 분석

## 1. 이슈 개요
- **요약**: 추출 모듈 메모리 임계치 초과 및 디스크 I/O 전이에 따른 성능 저하
- **우선순위**: 보통 (장애 전파 위험 있음)
- **분석 시점**: 2026-02-08

## 2. 기술적 원인 분석
- **영향도 카테고리**: 1. 자체 솔루션 결함 (추출 모듈)
- **상세 원인**:
  - `IN_MEMORY_TX_SIZE` 설정치(임계치)를 초과하여 캐시 파일로 전환되는 과정에서 I/O 병목 발생.
  - 아카이브 발생량이 평소와 다름없음에도 발생했다는 점, 기동 후 일정 기간(13일~50일) 경과 후 발생하는 점으로 미루어 볼 때 장기 기동 시 누적되는 메모리 관리 결함 가능성이 높음.

## 3. 재현 방법
- AIX 환경에서 대량 트랜잭션 혹은 장기 기동 테스트 필요. (현재 현장 로그 부족으로 특정 쿼리/트랜잭션 패턴 식별 불가)

## 4. 조치 제안
- **단기**: 주기적 모듈 재기동 스케줄링 가이드 (Workaround).
- **장기**: 추출 모듈의 메모리 프로파일링 및 AIX 환경에서의 메모리 할당/해제 로직 전수 조사.
