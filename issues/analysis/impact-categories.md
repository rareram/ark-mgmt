# 영향도 분류 기준

## 🎯 분류 목적
Jira 이슈의 근본 원인을 체계적으로 분류하여 각 스테이크홀더에게 적절한 수준의 보고를 제공하고, 반복 패턴을 식별하여 proactive한 대응이 가능하도록 한다.

## 📊 5대 영향도 카테고리

### 1. 자체 솔루션 결함 (Internal Solution Defect)
**정의**: AFC/AFO/AFF/TableDiff 솔루션 자체의 버그나 한계로 인해 발생한 이슈

**세부 분류**:
- **버그**: 코드 레벨의 명백한 오류
- **성능 문제**: 처리량, 지연 시간, 리소스 사용량 이슈
- **호환성 문제**: 특정 오라클 버전, OS, 환경과의 충돌
- **설계 결함**: 아키텍처나 설계의 근본적 한계

**판단 기준**:
- 동일 환경에서 재현 가능
- 다른 고객사에서도 동일 현상 보고
- 코드 분석 시 원인 명확히 확인
- 로그에 명백한 에러 메시지 존재

---

### 2. 고객사 환경 문제 (Customer Environment Issue)
**정의**: 고객사의 인프라, 네트워크, 시스템 구성으로 인해 발생한 이슈

**세부 분류**:
- **인프라 문제**: CPU, 메모리, 디스크 I/O 부족
- **네트워크 문제**: 방화벽, 대역폭, 연결 불안정
- **버전 호환성**: 지원되지 않는 오라클/OS 버전 사용
- **환경 설정**: 파라미터, 권한, 경로 설정 오류

**판단 기준**:
- 해당 고객사에서만 발생
- 환경 점검 시 명백한 문제 발견
- 테스트 환경에서는 정상 동작
- 환경 개선 시 현상 해소

---

### 3. 고객 담당자 행위 (Customer Operator Error)
**정의**: 고객사 담당자의 조작 미스, 설정 변경, 운영 실수로 인해 발생한 이슈

**세부 분류**:
- **조작 미스**: 잘못된 명령어 실행, 순서 오류
- **설정 오류**: 파라미터 잘못 입력, 권한 부여 실수
- **운영 미스**: 모니터링 부재, 정기 점검 누락
- **정책 위반**: 가이드라인 미준수, 비표준 운영

**판단 기준**:
- 고객사 담당자의 특정 행동 후 즉시 발생
- 행동 취소 시 현상 해소
- 동일 행동 반복 시 재현 가능
- 교육 시 문제 해결 가능

---

### 4. 애매/재현 불가 (Ambiguous/Non-reproducible)
**정의**: 원인을 특정할 수 없거나 재현이 불가능하여 명확한 분석이 어려운 이슈

**세부 분류**:
- **원인 불특정**: 로그 부족, 현상 일시 발생, 증상만 보고
- **재현 불가**: 특정 조건에서만 발생 but 재현 실패
- **정보 부족**: 고객사 정보 제공 한계, 망분리 제약
- **일시적 현상**: 한번만 발생 후 재현 안됨

**판단 기준**:
- 로그 분석으로 원인 파악 불가
- 다양한 시도로도 재현 실패
- 고객사 정보 제공 제약 존재
- 유사 사례 없음

---

### 5. 기타 (External/Others)
**정의**: 위 4가지 카테고리에 속하지 않는 외부 요인이나 제3자 솔루션 연동 문제

**세부 분류**:
- **외부 요인**: 타 솔루션, 외부 API, 네트워크 장애
- **제3자 연동**: 다른 벤더 솔루션과의 통합 문제
- **자연재해/재난**: 예외적 상황으로 인한 문제
- **보안 이슈**: 외부 공격, 바이러스, 보안 위협

**판단 기준**:
- 외부 시스템 장애와 시점 일치
- 제3자 솔루션 변경과 연관성
- 예외적 상황 발생
- 보안 관련 로그 발견

## 🔍 분석 프로세스

### Step 1: 정보 수집
- 이슈 발생 환경 파악
- 관련 로그 수집 및 분석
- 재현 시도 및 결과 확인
- 고객사 상황 파악

### Step 2: 카테고리 결정
- 위 판단 기준에 따른 평가
- 다중 카테고리일 경우 최우선 원인 선정
- 불확실 시 희종형님께 결정 질문

### Step 3: 조치 방안 수립
- 카테고리별 대응 템플릿 활용
- 스테이크홀더별 보고서 생성
- 재발 방지 대책 수립

## 📈 통계 활용
- 주간/월간 카테고리별 발생률 추이
- 반복 패턴 식별 및 proactive 대응
- 고객사별 문제 유형 분석
- 제품 개선 방향 도출
