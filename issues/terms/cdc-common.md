## 🔥 핵심 용어

### 기본 CDC 개념
| 용어 | 영문 | 정의 | 관련 솔루션 | 주의사항 |
|------|------|------|------------|----------|
| 캡처 | Capture | 소스 DB의 변경 데이터를 감지하는 과정 | AFC, AFO | 실시간성이 핵심 |
| 어플라이 | Apply | 캡처된 데이터를 타겟 DB에 반영하는 과정 | AFC, AFO | 순서 보장 중요 |
| 레플리케이션 | Replication | 소스에서 타겟으로 데이터를 복제하는 전체 과정 | AFC, AFO | 지연과 갭 관리 |
| 체인지 데이터 | Change Data | INSERT/UPDATE/DELETE 등의 변경된 데이터 | AFC, AFO | CDC의 대상 |

### 데이터 흐름
| 용어 | 영문 | 정의 | 관련 솔루션 | 주의사항 |
|------|------|------|------------|----------|
| 소스 | Source | 데이터가 발생하는 원본 데이터베이스 | AFC, AFO | 오라클 DB가 주로 해당 |
| 타겟 | Target | 데이터가 복제되는 대상 데이터베이스 | AFC, AFO | 다양한 DB 지원 |
| 파이프라인 | Pipeline | 데이터가 흐르는 전체 경로 | AFC, AFO | 병목 지점 발견 중요 |
| 큐 | Queue | 데이터를 일시적으로 저장하는 버퍼 | AFO | 용량 관리 필수 |

### 성능 및 지연
| 용어 | 영문 | 정의 | 관련 솔루션 | 주의사항 |
|------|------|------|------------|----------|
| 레이턴시 | Latency | 변경 발생부터 타겟 적용까지의 지연 시간 | AFC, AFO | SLA 지표 |
| 스루풋 | Throughput | 단위 시간당 처리할 수 있는 데이터 양 | AFC, AFO | 시스템 용량 척도 |
| 갭 | Gap | 처리되지 않고 누락된 데이터 구간 | AFC, AFO | 데이터 정합성 위협 |
| 병목 | Bottleneck | 전체 파이프라인의 처리량을 제한하는 지점 | AFC, AFO | 성능 튜닝 대상 |

### 일관성 및 무결성
| 용어 | 영문 | 정의 | 관련 솔루션 | 주의사항 |
|------|------|------|------------|----------|
| 컨시스턴시 | Consistency | 소스와 타겟 데이터의 일관성 상태 | AFC, AFO | 검증 필요 |
| 무결성 | Integrity | 데이터의 정확성과 완전성 보장 | AFC, AFO | 체크섬 활용 |
| 트랜잭션 순서 | Transaction Order | 소스에서 발생한 트랜잭션의 논리적 순서 | AFO | 순서 보장 중요 |
| 정합성 검사 | Validation Check | 소스-타겟 데이터 일치 여부 확인 | TableDiff | 주기적 실행 필요 |

## 🔄 운영 모드

### 복제 방식
| 용어 | 영문 | 정의 | 특징 |
|------|------|------|------|
| 실시간 복제 | Real-time Replication | 거의 실시간으로 데이터를 복제 | 낮은 레이턴시 |
| 배치 복제 | Batch Replication | 정해진 주기로 묶어서 복제 | 높은 스루풋 |
| 증분 복제 | Incremental Replication | 변경된 데이터만 복제 | 효율성 높음 |
| 전체 복제 | Full Replication | 전체 데이터를 복제 | 정합성 완벽 |

### 재해 복구
| 용어 | 영문 | 정의 | 사용 솔루션 |
|------|------|------|------------|
| 장애 조치 | Failover | 주 시스템 장애 시 예비 시스템으로 전환 | AFO |
| 복구 | Recovery | 장애 후 정상 상태로 복구 | AFC, AFO |
| 포인트인타임 | Point-in-time | 특정 시점으로 데이터 복구 | AFC |
| 롤백 | Rollback | 잘못된 변경을 이전 상태로 되돌리기 | AFC |

## 🚨 자주 혼동되는 용어

### Capture vs Extract
- **Capture**: 실시간 변경 감지 (CDC 방식)
- **Extract**: 정기적 데이터 추출 (ETL 방식)

### Latency vs Throughput  
- **Latency**: 개별 데이터 처리 속도 (응답성)
- **Throughput**: 전체 처리 능력 (용량)

### Gap vs Lag
- **Gap**: 데이터 누락 (질적 문제)
- **Lag**: 처리 지연 (시간적 문제)
