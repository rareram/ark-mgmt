# Jira 이슈 관리 시스템 - AGENTS.md

## 🎯 목적
SE와 개발팀 간의 **언어 장벽**을 해결하고, 파편화된 현장 정보와 코드 레벨의 추측을 연결하는 **'소통의 중심점'** 역할을 한다. 기술이사님이 각 스테이크홀더(임원, 영업, SE)에게 매번 번역해야 하는 수고를 줄이고, 객관적인 데이터를 기반으로 의사결정을 돕는다.

## 🏗️ 제품 및 환경 특성
- **AFC (CDC)**: 오라클 Redo 로그 역설계 기반 솔루션. 기술적 난이도가 높고 예외 케이스가 많음.
- **AFO (물리복제)**: 오라클 DB 물리적 복제 솔루션.
- **AFF (파일복제)**: 보조 로그 및 아카이브 로그 복제 (rsync 유사).
- **고객사 특성**: 관공서, 은행, 학교 등 보안에 매우 민감함. 장애 발생 시 상세 로그 수집이 극히 어려워 한정된 정보로 추론해야 함.

## 📋 분석 원칙 (이사님 가이드)
1. **정직한 추론**: AI가 억지로 원인을 꾸며내지 않는다. 정보가 부족하면 "정보 부족으로 알 수 없음"을 명확히 밝힌다.
2. **현상과 원인의 분리**: SE가 보고한 '증상'과 개발팀이 추측한 '근본 원인'을 명확히 구분한다.
3. **재현 여부 명시**: 증상 재현이 되는 것과 안 되는 것을 확실히 분리하여 보고한다.
4. **객관적 중재**: 어느 한 팀의 잘못을 탓하는 것이 아니라, 코드/환경/작업자 행위/일시적 현상 등 객관적인 범주로 분류한다.

## 🎯 핵심 기능

### 1. 용어사전 관리
- **실시간 등록**: 이슈 분석 중 새로운 용어 발견 시 자동 등록
- **분류 체계**: 오라클DB, CDC 통용, 회사 자체, SE 업무 용어로 분류
- **버전 관리**: 용어 정의 변경 시 히스토리 추적

### 2. 영향도 분석
이슈의 근본 원인을 5가지 카테고리로 분류:
- **자체 솔루션 결함**: 버그, 성능 문제, 호환성 이슈
- **고객사 환경 문제**: 인프라, 네트워크, 버전 호환성
- **고객 담당자 행위**: 설정 오류, 운영 미스
- **애매/재현 불가**: 원인 불특정, 일시적 현상
- **기타**: 외부 요인, 제3자 솔루션 연동 문제

### 3. 자동 보고서 생성
- **개발팀**: 기술적 원인 + 재현 방법 + 수정 제안
- **임원진**: 영향도 + 비즈니스 리스크 + 조치 계획  
- **영업팀**: 기술 배제 설명 + 고객 대응 가이드
