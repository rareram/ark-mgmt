# 보조 도구 (script)

TableDiff의 운영 편의성을 높이기 위해 제공되는 쉘 스크립트 모음입니다. 

**안내**
 
* 본 스크립트들의 버전(0.6.3)은 대상이 되는 **`TableDiff_0.6.3.jar` 파일과 호환성**을 맞추기 위해 동일하게 부여되었습니다.
* 각 스크립트 파일 내부에는 별도의 **리비전(Revision)** 정보가 `YYYYMMDD_HHmm` 형식(예: 20260126_1600)으로 기재되어 있어, 파일 내용을 열어보면 상세한 업데이트 시점을 확인할 수 있습니다.
* 이 스크립트들은 `bash` 쉘 환경에서 동작하며, 사용 전 실행 권한(`chmod +x`)이 필요합니다.
* 스크립트와 동일한 경로에 `TableDiff_0.6.3.jar` 파일이 있어야 하며 사전에 비교할 테이블 정보를 알고 있어야 합니다.

---

## 1. 설정 생성 스크립트

대화형 인터페이스를 통해 복잡한 HOCON 설정 파일을 손쉽게 생성합니다.

!!! warning "보안 및 접근 정책 준수 제약 사항"
    보안 사고 예방 및 접근 정책 준수를 위해, 본 스크립트는 **실제 DB에 접속하여 스키마를 조회하거나 컬럼명을 자동으로 추출하는 기능은 제공하지 않습니다.** 엔지니어가 직접 확인한 DB 정보와 SQL 컬럼 정보를 수동으로 입력해야 합니다.

* **다운로드**: [:material-download: 스크립트 다운로드](../../../../assets/scripts/gen_tablediffconf_0.6.3.sh)

```bash
# 사용 예시
chmod +x gen_tablediffconf_0.6.3.sh
./gen_tablediffconf_0.6.3.sh
```

??? info "상세 기능 (Feature)"
    1. **단계별 설정 마법사**
        - **버전 통일**: TableDiff 버전에 맞춰 향후 업그레이드로 사용 방법 또는 설정 파라미터가 변경되더라도 혼동 없이 사용 가능합니다.
        - **진행 단계 표시**: `Source DB 설정` -> `Target DB 설정` -> `Table Info 설정` -> `네이밍 설정` -> `Complete` 순서를 상단에 표시하여 진행 단계를 쉽게 파악할 수 있습니다.
    2. **DB 프로필 관리 및 재사용**
        - **최근 설정 로드**: `~/.tablediff_profile`에 소스/타겟 DB 접속 정보를 암호화(권한 600)하여 저장하고, 이를 기본값으로 활용하여 반복 작업을 효율화합니다.
    3. **이기종 DB 및 고급 옵션**
        - **멀티 DB 지원**: Oracle, PostgreSQL, MySQL, Tibero에 최적화된 JDBC URL 템플릿과 드라이버 정보를 제공합니다.
        - **정밀 비교 옵션**: 컬럼별 오차 허용 범위(Tolerance: Delta/Milli), 정렬 방향(Asc/Desc), Null 처리 방식 등 TableDiff의 고급 기능을 메뉴 선택만으로 설정에 반영할 수 있습니다.
    4. **ApplyTo 템플릿 자동 생성**
        - **컬럼 매핑**: PK와 일반 컬럼을 구분하여 JSON 결과 데이터를 DB에 반영하기 위한 `Change` 블록을 자동으로 구성합니다.
        - **Mock 모드 기본 구성**: 실수 예방을 위해 `Change` 블록의 기본 설정은 `Mock` 모드이며, 반영 SQL 미리보기가 가능합니다. (이후 주석 제거로 설정 변경 가능)
    5. **설정 파일 네이밍 및 기본 검증**
        - **Naming 컨벤션**: IP 기반, 별칭(Alias) 기반, 테이블명 기반 등 파일을 열어보지 않아도 식별하기 좋은 이름으로 생성하는 옵션을 제공합니다.
        - **설정 생성 후 검증**: 파일 생성 직후 HOCON 포맷(중괄호 쌍, 따옴표, 필수 키값 누락 여부)을 즉시 검사하여 무결성을 보장합니다.

### 1.1. [옵션] 스키마 조회 스크립트

기본 설정 생성 스크립트(`gen_tablediffconf`)는 보안 정책 준수를 위해 DB 직접 접속을 배제하고 있으나, **접근 제어가 확보된 환경(사내망, 개발망 등)**에서 다수의 컬럼을 가진 테이블을 설정할 때의 편의성을 위해 **스키마 정보 조회 전용 스크립트**를 선택적으로 제공합니다.

* **목적**: DB에 원격 접속하여 대상 테이블의 컬럼 및 PK 정보를 조회하고, 이를 포맷팅하여 제공함으로써 수동 입력 시의 오타 방지 및 시간 절약.
* **파일명**: `sch_tablediff_0.6.3.sh`
* **다운로드**: [:material-download: 스크립트 다운로드](../../../../assets/scripts/sch_tablediff_0.6.3.sh)

!!! check "사전 요구 사항 (Prerequisites)"
    본 스크립트는 DB 접속을 위해 로컬 환경에 **Native Client**(`sqlplus`, `psql`, `mysql`)가 설치되어 있거나, **Docker** 컨테이너를 통한 우회 접속이 가능한 환경이어야 동작합니다.

## 2. 실행 보조 스크립트

JVM 메모리 자동 튜닝 및 간편한 메뉴 인터페이스를 제공하는 실행 래퍼(Wrapper)입니다.

* **파일명**: `run_tablediff_0.6.3.sh`
* **주요 기능**: 메모리 자동 설정, Compare/Apply 모드 선택, 실행 결과 요약
* **다운로드**: [:material-download: 스크립트 다운로드](../../../../assets/scripts/run_tablediff_0.6.3.sh)

```bash
# 사용 예시
chmod +x run_tablediff_0.6.3.sh

# 일반 모드 실행
./run_tablediff_0.6.3.sh

# 개발자 모드 실행 (메모리 오버라이드, 프로파일러 연동 등)
./run_tablediff_0.6.3.sh -dev
```

??? info "상세 기능 (Feature)"
    1. **실행 환경 자동 설정**
        - **Heap Memory 자동 최적화**: 시스템 전체 메모리를 감지하여 60%를 할당(최소 1GB ~ 최대 8GB)하며, 별도 JVM 옵션 설정 없이 최적화된 파라미터를 자동 입력합니다.
        - **Java 환경 검증**: `JAVA_HOME` 및 경로 내 Java 실행 파일과 버전을 자동으로 체크하여 구동 가능 여부를 사전 점검합니다.
    2. **대화형 모드 관리**
        - **Compare 모드**: `.conf` 파일 리스트에서 선택하여 테이블 비교를 수행하고 결과를 NDJson 파일로 생성합니다.
        - **Apply 모드**: 생성된 JSON 결과 파일을 선택하여 실제 DB에 변경 사항(Insert/Update/Delete)을 반영합니다.
        - **List Runs 모드**: 현재 디렉토리에 존재하는 과거 작업 결과(`result_*.json`)와 변경 건수 요약을 한눈에 조회합니다.
    3. **구성 설정 (conf) 사전 검증**
        - **Integrity Check**: 실행 전 HOCON 설정의 괄호 쌍(`{ }`), 따옴표(`" "`), 필수 키 값(Driver, URL, Table 명 등) 누락 여부를 검사하여 런타임 에러를 예방합니다.
    4. **운영 안전 장치 (Safety First)**
        - **Apply Plan 미리보기**: 실제 반영 전, 설정 파일에 정의된 액션(Action), 대상 테이블, 배치 크기 등을 요약 출력합니다.
        - **Risk Labeling**: 설정 내용을 분석하여 작업의 위험도(High/Medium/Low Risk 또는 Dry-Run)를 명시적으로 표시합니다.
        - **최종 승인 토큰**: 실수 방지를 위해 `APPLY`라는 키워드를 직접 입력해야만 DB 반영이 시작됩니다.
    5. **작업 추적 및 사후 감사 (Audit & Logging)**
        - **추적용 stdout 로그 생성**: 모든 작업에 실행 시점의 사용자, 호스트, OS 정보, JVM 옵션, 실행 명령어를 포함한 전용 로그 파일을 생성합니다.
        - **실행 요약 보고**: 작업 완료 즉시 소요 시간, 종료 코드, 변경 데이터 건수(Same, Changed, Only in A/B)를 화면에 리포트합니다.

## 3. 결과 확인 스크립트

생성된 비교 결과 파일(`result_*.json`)들을 스캔하여 작업 이력 및 상세 분석 리포트를 제공합니다.

!!! note "사용 환경 및 제약 사항"
    DB 작업의 특성상 여러 경로를 동시에 참조할 경우 발생할 수 있는 **운영 실수 및 환경 혼동을 방지**하기 위해, 본 스크립트는 **현재 실행 경로(Current Directory) 내의 파일만 탐색**하도록 설계되었습니다. 프로젝트별로 폴더를 나누어 관리 중이라면 해당 폴더로 스크립트를 복사하여 사용하십시오.

* **파일명**: `chk_tablediff_0.6.3.sh`
* **주요 기능**: 작업 일시별 결과 목록 조회, 상태(MATCH/DIFF/FAIL) 확인, 상세 통계 리포트 출력
* **다운로드**: [:material-download: 스크립트 다운로드](../../../../assets/scripts/chk_tablediff_0.6.3.sh)

```bash

# 사용 예시

chmod +x chk_tablediff_0.6.3.sh

./chk_tablediff_0.6.3.sh

```



??? info "상세 기능 (Feature)"

    1. **작업 타임라인**

        - **최신순 정렬**: 현재 디렉토리 내의 모든 결과 파일(`*.json` + `*.json.log`)을 스캔하여 가장 최근에 수행된 작업부터 리스트 형태로 출력합니다.

        - **상태 요약 대시보드**: 각 작업 상태를 `MATCH`(일치), `DIFF`(불일치), `FAIL`(실패)로 구분하여 표시하며, 변경된 레코드 수를 한눈에 확인할 수 있습니다.

    2. **상세 감사 및 결과 리포트**

        - **작업 컨텍스트 복원**: 실행 당시의 작업자(User), 호스트(Host), 사용된 설정 파일, Java 버전 및 Heap 메모리 정보 등을 로그에서 추출하여 리포트합니다.

        - **데이터 정합성 판정**: 단순 수치 나열이 아닌 데이터 일치 여부를 기반으로 `INTEGRITY CONFIRMED`(정합성 확인) 또는 `DISCREPANCIES FOUND`(차이 발견) 등을 최종 판정 결과로 제공합니다.

    3. **실패 작업 분석**

        - **로그 스캔**: 실패(FAIL)한 작업의 경우, 엔지니어가 직접 로그를 열지 않아도 스크립트가 주요 예외(Exception)나 에러 메시지를 추출하여 화면에 표시합니다.

        - **빠른 원인 파악**: `Caused by` 구문을 추적하여 작업 실패의 근본 원인을 상단에 노출합니다.

    4. **파일 정보**

        - **결과 파일 추적**: 리스트에서 선택한 작업의 실제 JSON 파일 경로와 파일 크기를 표시하여 데이터 용량을 확인합니다.

        - **마지막 파일 표시**: 스크립트 종료 시 마지막으로 확인한 파일명을 표시하여 CLI 환경에서 후속 작업을 이어갈 수 있도록 돕습니다.
