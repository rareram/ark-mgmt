# OCI 운영자 가이드

이 문서는 Oracle Cloud Infrastructure (OCI)의 인프라를 관리하는 **운영자**를 위한 핵심 가이드입니다.

운영자는 OCI 환경의 안정성, 보안, 비용 효율성을 책임지며, 모든 활동은 아래 가이드라인을 따라야 합니다. 이 페이지는 운영자가 알아야 할 전체 문서 구조를 안내하는 길잡이 역할을 합니다.

---

## 1. 운영자의 핵심 책임

- **리소스 및 비용 관리**: 모든 리소스는 라이프사이클에 따라 관리되어야 하며, 비용 최적화를 항상 고려해야 합니다.
- **보안 및 권한 관리**: 최소 권한 원칙에 따라 사용자의 접근 권한을 관리하고, 보안 정책을 준수해야 합니다.
- **안정적인 서비스 운영**: 인프라의 장애를 방지하고, 모니터링 및 자동화를 통해 안정적인 서비스를 제공해야 합니다.

## 2. 상세 가이드 목록

각 주제에 대한 상세한 내용은 아래의 해당 가이드를 참고하십시오.

- **[거버넌스 프레임워크](./governance-framework.md)**
    - OCI 운영의 전체적인 원칙, 정책, 책임 모델을 정의하는 최상위 문서입니다. **운영자는 반드시 가장 먼저 읽어야 합니다.**

- **[멀티테넌시 아키텍처](./multi-tenancy-architecture.md)**
    - 테넌시의 논리적 구조, 구획(Compartment) 및 도메인(Domain) 분리 모델을 설명합니다.

- **[IAM 상세 가이드](./iam-guide.md)**
    - 신규 입사자 계정 생성, 그룹 및 역할 기반의 권한 부여 방법(Policies)을 안내합니다.

- **[보안 가이드](./security-guide.md)**
    - 네트워크(VCN, NSG), 데이터베이스 접근 제어, 로깅 및 감사 등 핵심 보안 수칙을 다룹니다.

- **[비용 관리 가이드](./cost-management.md)**
    - 예산 설정, 비용 분석, 지출 최적화 방법에 대해 안내합니다.

- **[자동화 가이드](./automation-guide.md)**
    - 반복적인 작업을 자동화하고(IaC, 스크립트), 운영 효율을 높이는 방법을 소개합니다.

- **[관리자 계정 목록](./admin-accounts.md)**
    - 공용으로 사용하는 관리자 계정의 종류와 용도를 설명합니다.
