# OCI 비용 및 지출관리 가이드

이 문서는 OCI 환경의 비용을 효과적으로 추적, 분석, 최적화하기 위한 가이드입니다.

## 1. 예산 모니터링

월별 사용 목표를 설정하고, **OCI Budgets 기능**을 이용해 예산 초과를 사전에 방지하는 것이 중요합니다.

- **설정**: 컴파트먼트 또는 태그 단위로 예산을 설정합니다.
- **알림**: 예산의 특정 비율(예: 80%, 100%)에 도달했을 때 지정된 담당자에게 알림(Email, Slack 등)이 가도록 구성합니다.
- **강제 중단**: Budgets 기능 자체는 알림을 제공할 뿐, 리소스 생성을 강제로 막지는 않습니다. 예산 초과 시 리소스 생성을 중단시키려면, Quota 정책과 OCI Functions를 결합한 별도의 자동화 스크립트 구현이 필요합니다.

**권장 사항**: 모든 주요 컴파트먼트에 Budget을 설정하고, 비용 책임자(Owner)를 알림 수신자로 지정하십시오.

## 2. 비용 분석 및 인과관계 추적

**OCI Cost Analysis** 기능은 비용이 발생하는 위치와 원인을 파악하는 가장 중요한 도구입니다.

- **시각화**: 서비스별, 컴파트먼트별, 태그별 비용 내역을 시각화하고 분석합니다.
- **정기 리포트**: 매월 **"가장 비용이 많이 발생한 리소스 TOP 5"** 리포트를 작성하고, 다음 항목을 중점적으로 검토합니다.
    a. **오버프로비저닝**: 실제 사용량에 비해 과도하게 큰 컴퓨트 Shape이나 스토리지가 할당되지 않았는가?
    b. **유휴 자원 (Idle Resources)**: 사용하지 않는(Stopped) 인스턴스, 연결되지 않은(Detached) 블록 볼륨, 비어있는 LB 등이 계속 비용을 유발하고 있지는 않은가?
    c. **미종료 테스트 리소스**: 완료된 프로젝트나 테스트의 리소스가 삭제되지 않고 남아있지는 않은가?
- **인과 분석**: "이 비용이 왜 발생했는가?"를 팀별로 리뷰하고, 비용 최적화 방안을 논의합니다.

## 3. 자동화된 리소스 관리

비용 낭비를 막기 위한 자동화된 정책을 구현합니다.

- **리소스 스케줄링**: 개발/테스트 환경의 인스턴스는 업무 시간(예: 09:00-19:00) 외에는 자동으로 중지되도록 스케줄링합니다.
- **유휴 자원 정리**:
    - 특정 기간(예: 2주) 이상 `Stopped` 상태인 인스턴스는 담당자에게 알림 후 삭제 대상으로 분류합니다.
    - 오랫동안 `Detached` 상태인 블록 볼륨이나 로드밸런서를 탐지하여 정리하도록 체크리스트를 구성하고 자동화 스크립트를 적용합니다.

## 4. 비용 책임 배분 및 내부 청구

- **태그 기반 비용 분리**: 리소스에 적용된 `Project`, `Team` 등의 태그를 기준으로 "어떤 팀이 얼마를 썼는지"를 명확하게 분리합니다.
- **내부 책임**: 태그 기반 비용 분석 결과를 내부 회계 또는 프로젝트 원가 산정에 반영합니다.
- **월간 리뷰**: 각 팀의 리더 또는 프로젝트 책임자에게 월간 비용 리포트를 공유하고, 비용 사용 계획 및 최적화 방안에 대해 논의합니다.
